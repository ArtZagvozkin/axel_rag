# axel_rag

Прототип **интеллектуального ассистента для корпоративной базы знаний AxelNAC** на базе подхода **Retrieval-Augmented Generation (RAG)**.

Проект покрывает ключевые этапы RAG-пайплайна:
- подготовка и хранение корпуса (PostgreSQL)
- индексация и загрузка векторных представлений (Qdrant)
- утилиты для наполнения/обновления индекса
- (опционально) слой приложения/сервисов для последующего QA/бота/HTTP API

## Архитектура
1. **Корпус** хранится в PostgreSQL (статьи и чанки).
2. Для каждого чанка строится **эмбеддинг** (например, `intfloat/multilingual-e5-small`).
3. Эмбеддинги и метаданные чанков загружаются в **Qdrant** (коллекция, например `axel_chunks`).
4. На базе Qdrant далее можно строить retrieval/ranking и генерацию ответов LLM.

## Структура репозитория
- `db/` — скрипты/инициализация БД/векторной коллекции.
- `util/` — утилиты индексации и обслуживания пайплайна (загрузка чанков из Postgres в Qdrant батчами).
- `services/` — сервисные конфиги/обвязка (docker-compose, systemd, конфиги инфраструктуры).
- `app/` — код приложения (API/бот/интерфейс), который использует retrieval из Qdrant.
